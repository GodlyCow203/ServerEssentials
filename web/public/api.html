Overview

ServerEssentials provides a modular API system for Minecraft server management. Each API module is designed for thread-safe, asynchronous operations with comprehensive event support. Click any module below to view detailed documentation, usage examples, and implementation guides.

Implementation Guidelines

All ServerEssentials APIs follow these core principles:

  • Thread Safety: All methods are thread-safe and return CompletableFuture for async operations
  • Event-Driven: Comprehensive event system for monitoring all state changes
  • Null Safety: All public methods use @NotNull annotations
  • Immutability: Data classes are immutable with defensive copying
  • Economy Integration: Built-in support for Vault and internal economy systems
  • Persistent Storage: Automatic saving to disk/database with reload support

Getting Started

All APIs can be accessed through the ServerEssentials plugin manager:

ServerEssentialsAPI api = ServerEssentials.getAPI();
EconomyAPI economy = api.getEconomy();
WarpAPI warps = api.getWarp();

Refer to individual API documentation for detailed usage examples and configuration options.
